# Styling and UI Development Prompts

## Theme System Implementation
```
Create a comprehensive theming system with:
- Light and dark theme color palettes
- Typography scale and font families
- Spacing and sizing system
- Border radius and shadow utilities
- Color semantic tokens (primary, secondary, success, error, warning)
- Theme context provider with toggle functionality
- Theme persistence using AsyncStorage
```

## Component Styling
```
Style all components with:
- Consistent spacing using the spacing system
- Proper color usage from theme tokens
- Responsive design considerations
- Platform-specific styling (iOS/Android differences)
- Accessibility-friendly color contrasts
- Proper text scaling support
- RTL language support preparation
```

## Layout System
```
Implement a robust layout system:
- Flexbox-based layouts with proper alignment
- Safe area handling for notched devices
- Responsive breakpoints for different screen sizes
- Grid system for product displays
- Proper keyboard avoidance handling
- Scroll behavior and loading states
```

## Animation and Transitions
```
Add smooth animations and transitions:
- Cart item add/remove animations
- Page transitions between tabs
- Modal open/close animations
- Loading skeleton screens
- Button press states
- List item hover/focus states
- Theme transition animations
```

## Icon System
```
Create an icon system with:
- Consistent icon library (react-native-vector-icons)
- Icon size and color standards
- Custom SVG icons for brand elements
- Icon accessibility with proper labels
- Icon loading and caching
- Themed icon colors
```

## Form Elements
```
Design and implement form elements:
- Custom input fields with validation
- Search input with clear button
- Number inputs for quantity
- Toggle switches for settings
- Select dropdowns for categories
- Radio buttons and checkboxes
- Form error states and messages
```

## Visual Hierarchy
```
Establish clear visual hierarchy:
- Proper heading levels and sizes
- Content grouping and sectioning
- Call-to-action button prominence
- Information density management
- White space usage for readability
- Focus states and interactive feedback
```

## Platform-Specific Design
```
Adapt design for iOS and Android:
- iOS-style navigation and controls
- Material Design components for Android
- Platform-appropriate animations
- Native gesture support
- Platform-specific fonts and spacing
- Status bar and navigation bar handling
```

## Accessibility Implementation
```
Ensure accessibility compliance:
- Screen reader support with proper labels
- High contrast mode support
- Large text scaling support
- Focus management for keyboard navigation
- Color blind friendly palette
- Touch target size compliance
- Semantic HTML structure where applicable
```
